21. Frequently Asked Interview Questions
Q1. What is a Race Condition?

When multiple threads modify shared data simultaneously causing inconsistent results.

Q2. Why do we need synchronization?

To protect critical sections and avoid race conditions.

Q3. Is synchronized thread-safe?

Yes.

Q4. Is synchronized reentrant?

Yes.

Q5. Does synchronized use ReentrantLock?

No.

It uses Java's intrinsic monitor lock, which is reentrant.

Q6. Difference between synchronized and Lock?

synchronized is implicit and simpler.

ReentrantLock is explicit and offers advanced features such as:

tryLock()
Fairness
Interruptibility
Q7. What is a Monitor?

The internal lock associated with every Java object.

Q8. What is Deadlock?

When two or more threads wait forever for each other's locks.

Q9. What is volatile?

Provides visibility of changes across threads but does not prevent race conditions.

Q10. What is AtomicInteger?

A lock-free thread-safe integer implementation.

ReentrantLock → Explicit lock with advanced control.

Deadlock → Threads waiting forever for each other.

volatile → Visibility only.

ReadWriteLock → Multiple readers, single writer.
